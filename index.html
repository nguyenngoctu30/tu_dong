Hướng Dẫn: Luồng Truyền Dữ Liệu ESP32 & Giao Diện
1
Đọc Cảm Biến

Mạch ESP32 đọc thông số từ DHT22, MQ2 và module định vị GPS định kỳ 15 giây.

2
Gửi (Publish)

ESP32 đẩy dữ liệu qua mạng WiFi lên MQTT Broker broker.emqx.io (Port 1883).

3
Nhận (Subscribe)

Trang Web mở kết nối WebSockets (Port 8084) để lắng nghe các Topic tương ứng (VD: /temperature).

4
Cập Nhật & Điều Khiển

JavaScript xử lý dữ liệu để cập nhật số liệu/bản đồ. Người dùng có thể bấm nút trên Web để gửi lệnh ngược lại ESP32.

Tài Liệu: Danh Sách MQTT Topics
Tiền tố (Prefix): tuiot/freezer/
Tên Topic Chiều Truyền Dữ Liệu (Payload) Chức Năng & Ý Nghĩa
temperature ESP32 Web 25.50 (Số thực) Cập nhật nhiệt độ hiện tại từ cảm biến DHT22 (15s/lần).
humidity ESP32 Web 60.20 (Số thực) Cập nhật độ ẩm hiện tại từ cảm biến DHT22 (15s/lần).
gas ESP32 Web 1 hoặc 0 Trạng thái rò rỉ khí gas từ cảm biến MQ2 (1 = Cảnh báo).
gps ESP32 Web 10.82,106.62 Tọa độ Vĩ độ, Kinh độ lấy từ module GPS để vẽ lên bản đồ.
fan_status ESP32 Web ON hoặc OFF Báo cáo trạng thái đang hoạt động thực tế của Quạt.
alert ESP32 Web Cảnh báo... Gửi thông điệp khẩn cấp để hiển thị Popup (Swal) lên web.
fan_mode Web ESP32 AUTO / MANUAL Lệnh chuyển đổi chế độ hoạt động của Quạt.
fan_ctrl Web ESP32 ON hoặc OFF Lệnh ép bật/tắt Quạt (Chỉ tác dụng khi ở chế độ MANUAL).
threshold Web ESP32 22.5 (Số) Lưu ngưỡng nhiệt độ báo động vào EEPROM của ESP32.
phone Web ESP32 +84... (Chuỗi) Cập nhật số điện thoại cảnh báo vào mạch để gọi bằng SIM.
test_call Web ESP32 CALL Lệnh yêu cầu mạch ESP32 thực hiện gọi điện test ngay lập tức.
